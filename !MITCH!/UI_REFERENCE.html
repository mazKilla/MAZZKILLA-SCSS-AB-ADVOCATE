Section 04

Quick Glossary

Viewport
The visible area of a web page in the browser window. Everything outside it is scrolled out of view. On mobile, the viewport is the physical screen. Used in phrases like "fixed to the viewport" (always visible regardless of scroll).
Breakpoint (mobile / tablet / desktop)
A specific screen width at which the layout changes. Common breakpoints: Mobile (<768px), Tablet (768вҖ“1024px), Desktop (>1024px). When you say "fix this on mobile", you mean below the mobile breakpoint.
Responsive vs Adaptive
Responsive: The layout fluidly adjusts to any screen width using flexible units (%, vw, CSS Grid/Flex). One codebase, infinite screen sizes. Adaptive: Distinct fixed layouts are served at specific breakpoints вҖ” basically different designs for mobile/tablet/desktop.
z-index / Stacking Order
Controls which overlapping elements appear on top. Higher z-index = closer to the user (rendered in front). Modals typically have a very high z-index (e.g., 1000) so they appear above all other content. If something is appearing behind something else unexpectedly, it's usually a z-index problem.
Fixed vs Absolute vs Relative vs Sticky
CSS position values: Relative вҖ” offset from where the element normally sits, still takes up space. Absolute вҖ” removed from flow, positioned relative to nearest positioned ancestor. Fixed вҖ” positioned relative to the viewport, stays put on scroll. Sticky вҖ” normal flow until a scroll threshold, then acts like Fixed.
Flex vs Grid
Flexbox (display:flex): One-dimensional layout вҖ” arranges items in a row OR a column. Great for navbars, button rows, centering a single element. CSS Grid (display:grid): Two-dimensional layout вҖ” rows AND columns simultaneously. Great for full page layouts and card grids.
Padding vs Margin
Padding: Space inside an element between its content and its border. Clicking inside padding still triggers the element. Margin: Space outside an element's border, between it and neighboring elements. Think: padding is inside the box, margin is outside the box.
Overflow: hidden / scroll / auto
What happens when content is larger than its container. hidden вҖ” clips/cuts off overflowing content (nothing is scrollable). scroll вҖ” always shows scrollbars. auto вҖ” only shows scrollbars when content actually overflows (most common). If content is getting cut off, overflow is likely set to hidden.
opacity vs visibility vs display:none
opacity:0 вҖ” invisible but still takes up space and is still clickable. visibility:hidden вҖ” invisible and not clickable, but still takes up space. display:none вҖ” completely removed from layout. Takes up no space. Not clickable. Screen readers skip it. When you say "hide this", a developer needs to know which behavior you want.
State (hover, active, focus, disabled)
Visual states an interactive element can be in: hover вҖ” mouse is over it. active вҖ” currently being clicked/pressed. focus вҖ” selected via keyboard Tab key (usually shows an outline). disabled вҖ” cannot be interacted with (greyed out). Each state can have different styles.
Props (in React context)
Short for "properties." Values passed into a React component to configure it. Like arguments to a function. Example: a Button component might accept props like label="Submit", disabled={true}, onClick={handleSubmit}. When a developer says "pass a prop", they mean pass a value into a component.
Component
A reusable, self-contained piece of UI built in code (React, Vue, etc.). A Button, a Card, a Modal, a Navbar are all components. They can be composed together to build pages. When you say "add a component", you're asking for a reusable building block to be created or used.
Render
The process of converting code/data into visible HTML on screen. "Re-render" means the component updates its display in response to new data or a user action. "It's not rendering" means it's not appearing on screen. "It re-renders too often" means it's refreshing more than necessary, causing performance issues.
Hardcoded vs Dynamic
Hardcoded: A value is written directly in the code and cannot change without a code edit. Example: the text "Welcome, Mitchell" written literally in the HTML. Dynamic: The value comes from data (a variable, database, API). Example: "Welcome, {username}" where the name changes per user. When you say "make this dynamic", you mean connect it to real data.
